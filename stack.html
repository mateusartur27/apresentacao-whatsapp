Trade-offs — Por que escolheram X e não Y?

Backend — Mensageria

✅ Erlang Processos leves (KB cada), pattern matching nativo, links de supervisão, hot code upgrade. Ideal para millhões de conexões persistentes.
vs Java JVM tem GC pauses. Threads são pesadas (1MB stack). Latência p99 maior. Não foi adotado.
vs Go Go goroutines são boas, mas o WhatsApp já era Ejabberd/Erlang desde 2009 e não há motivo para migrar.

Banco de Dados — Fila Offline

✅ ScyllaDB C++, latência p99 <5ms, compatível com Cassandra mas 10x mais rápido, sem GC Java.
vs Cassandra Mesma API CQL, mas GC stop-the-world do Java causava spikes de latência inaceitáveis.
vs MySQL SQL relacional não escala horizontalmente para bilhões de mensagens write-heavy.

Criptografia E2E

✅ Signal Protocol Double Ratchet + X3DH = forward secrecy + break-in recovery. Open source, auditado, battle-tested.
vs PGP PGP não tem forward secrecy. Comprometer chave privada expõe todo histórico. Não adequado para chat.
vs TLS somente TLS criptografa o canal, mas o servidor ainda vê o conteúdo. E2E significa nem o WhatsApp vê.

Chamadas — WebRTC

✅ WebRTC P2P Latência mínima — stream vai direto entre devices. Sem custo de servidor para o tráfego de áudio/vídeo quando em P2P.
vs SFU sempre Um SFU (Selective Forwarding Unit) é usado apenas em chamadas em grupo (>4 pessoas), pois P2P escalaria mal.
vs VoIP SIP SIP puro requereria infraestrutura de operadora e não tem E2E nativamente como WebRTC + DTLS-SRTP.

Evolução Tecnológica — WhatsApp ao longo do tempo

2009
Fundação + Erlang/XMPP
Jan Koum e Brian Acton escolhem Erlang/Ejabberd para o servidor de mensagens. Stack inicial: Erlang, FreeBSD, SQLite nos clientes.
2011
1 milhão de usuários — Scale inicial
Mnesia como banco nativo Erlang para persistência de sessões e routing. HAProxy para load balancing. FreeBSD tuning de kernel.
2014
Chamadas de Voz (aquisição pela Meta)
Adição de VoIP baseado em WebRTC. Adoção de CoTURN. Meta adquire WhatsApp por US$19 bilhões. Início da integração de infra Meta.
2016
E2E em Todas as Mensagens (Signal Protocol)
Parceria com Open Whisper Systems. Implementação do Signal Protocol com X3DH + Double Ratchet. Toda comunicação passa a ser E2E criptografada.
2016
WhatsApp Web + Desktop
Lançamento do cliente Web (React + WebAssembly para libsignal). Cliente Desktop com Electron. Protocolo único WebSocket para todos os clientes.
2018
Chamadas de Vídeo em Grupo (SFU)
Adição de Selective Forwarding Unit para chamadas em grupo. ScyllaDB substitui Cassandra para offline message queue. Memcached scale-out.
2021
Multi-Device e Linked Devices
Arquitetura de multi-device sem depender do telefone como hub. Sincronização de histórico criptografado entre dispositivos via protocolo de identidade de dispositivo.
2023–24
WhatsApp Channels + IA + QUIC
Adição de Canais (broadcast público). Experimentos com HTTP/3 + QUIC para reduzir handshake de conexão. Integração de Meta AI. Backups E2E opcionais na nuvem.
